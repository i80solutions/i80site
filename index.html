The platform

What is i80agent?

i80agent is a platform for building Knowledge Agents from private domain knowledge. A Knowledge Agent can use generic retrieval to access trusted domain memory and, when the domain requires expertise, use structured intent to guide specialized LLM workflows.

Personal Examples

Retrieval

Cooking stories, memories, notes, decisions, creative process, and lived experience.

Intent

Plan a dinner menu, evaluate a creative idea, or turn past notes into a guided recommendation.

Business Examples

Retrieval

Hotel guest support, HR onboarding, internal help desks, product documentation, service FAQs, and policy lookup.

Intent

Review risks, compare policies, triage requests, evaluate documents, or generate structured recommendations.

Retrieval + Intent

Two Capabilities, One Agent Platform

Not every domain needs the same kind of agent. Some agents mainly need generic retrieval from trusted knowledge. Others need expert intent processing for review, analysis, decisions, or recommendations. More advanced systems can coordinate both.

Retrieval

Generic Knowledge Retrieval

For known questions, the agent retrieves answers directly from your structured knowledge base. If confidence is high, it can respond instantly without calling an LLM.

  • Semantic search and direct retrieval
  • Fast, low-cost, and grounded in trusted content
  • LLM optional when polishing, explaining, or handling ambiguity

Example: “What time is check-out?”

Intent

Expert Intent Processing

For complex requests, the agent selects a predefined expert intent, loads the right workflow, and uses an LLM to perform specialized processing.

  • Reusable task frameworks for review, analysis, and decision support
  • Always LLM-driven, but guided by domain expertise encoded as intent
  • Designed for domains where the answer is not already stored

Example: “Review this report and identify risks.”

Knowledge Agents can access domain memory, apply expert intent, or coordinate both.

Why not just ChatGPT?

LLMs are powerful, but without trusted domain retrieval and expert intent, they are incomplete. i80agent grounds both knowledge lookup and specialized workflows in your own domain.

Why domain-specific?

A hotel, restaurant, product, team, clinic workflow, or personal archive each has its own rules, vocabulary, context, answers, and tasks. The agent should know that world deeply.

Why build in stages?

A Knowledge Agent does not need a perfect knowledge base or intent workflow on day one. Whether it uses retrieval, intent, or both, it should improve from real usage and feedback.

How it works

Retrieval, intent, or both

i80agent supports different Knowledge Agent patterns. Some agents only need memory access. Some need expert processing. More advanced systems can coordinate both through orchestration.

1

Structure Knowledge

Structure domain knowledge for retrieval, and capture expert intent for use cases that need guided workflows and LLM processing.

2

Retrieval Agent

Access domain memory through embeddings, semantic search, and grounded answer retrieval.

3

Intent Agent

Apply domain expertise through structured workflows, expert templates, and controlled LLM processing.

4

Orchestrator Agent

Coordinate retrieval agents, intent agents, and external workflows when a domain needs a composable system.

Development Journey

Current State & Progress

The system is working — but still evolving.

What started as a personal experiment has grown into a broader exploration of how Knowledge Agents can be built, refined, and eventually made accessible to others. I am building this in my spare time, so progress comes in waves and moves more slowly than I would like.

Progress So Far

October 2024

Built the first working agent based on my personal cooking stories.

  • Built manually with AI assistance
  • Ran locally
  • Structured around memory, retrieval, and answer generation
  • Focused on capturing personal knowledge rather than generic recipes
December 2024

Extended the same architecture to a real-world use case:

  • Hotel Assistant Agent deployed at Courtyard Shanghai International Tourism and Resorts Zone
  • Validated that domain-specific agents can work in production environments
February 2025

Rebuilt the Alex Cooking agent into the Spontaneous Cooking by Alex Set Menu dinner experience, running it locally to guide and enrich dinners with guests.

  • Adapted the personal cooking knowledge base into a focused, guest-facing dinner experience centered on the set menu
  • Used the agent locally during live dinner settings
  • Explored how personal memory and storytelling could support a real-world hospitality experience
2025 – Refining the Hotel Assistant

Continued improving the hotel knowledge base with customer feedback and real usage patterns.

  • Refined hotel-specific knowledge based on customer questions and feedback
  • Improved the UI flow for a clearer guest assistant experience
  • Iterated on answer quality, coverage, and practical usability
January - March 2026 – Reflection & Reset

As AI evolved rapidly, I paused active development to rethink the foundation:

  • How should a generic knowledge system be designed?
  • What makes an agent reusable across domains?
  • How can this be built without heavy engineering overhead?

This led to a new direction: rebuilding i80agent using tools like Claude Code and OpenAI Codex, with the goal of minimizing manual coding and focusing on system design instead.

Starting April 2026

Began rebuilding the i80agent platform with ChatGPT, OpenAI Codex, Claude Chat, and Claude Code, using AI-assisted development to move faster from system design to working code.

May 2026

Rebuilt the i80.com website using OpenAI Codex without writing a single line of code manually, and refined the platform concept around Retrieval Agents, Intent Agents, and Orchestrator Agents.

What Works Today

  • Knowledge-based retrieval grounded in structured content
  • Query-focused embedding strategy for better matching
  • Domain-specific answer cards aligned with real user questions
  • Basic orchestration logic for direct answers, LLM use, and fallback
  • Early design direction for expert intent workflows

What Is Still Manual

  • Agent setup and configuration
  • Knowledge base structuring and ingestion
  • Publishing and deployment
  • Use-case tuning and iteration

What I’m Working on Next

The goal is to evolve i80agent into a generic platform for building Knowledge Agents that can work as Retrieval Agents, Intent Agents, or Orchestrator Agents depending on the domain and use case.

Current focus areas:

  • A simpler, more intuitive agent creation flow
  • Better knowledge management UI
  • Confidence-based routing and decision logic
  • Reusable expert intent templates for review, analysis, and decision support
  • Multi-domain scalability
  • A path from retrieval and expert workflows to triggered actions (APIs, workflows, integrations)
